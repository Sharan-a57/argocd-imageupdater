(((Running the python app)))
venv/bin/python app.py

(((Argocd Installation)))
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

(((PortForwarding)))
kubectl port-forward svc/argocd-server -n argocd 8080:443

(((Admin Password)))
kubectl get secret argocd-initial-admin-secret -n argocd -o jsonpath="{.data.password}" | base64 --decode && echo

(((Access the application)))
minikube service flask-app-service -n dev
(OR)
kubectl port-forward svc/flask-app-service 5000:5000 -n dev
Open application on Localhost:5000

(((ImageUpdater installation)))
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/stable/manifests/install.yaml

Create a k8s directory and add all of the deployment, svc and cm files with a kustomize file, why kustomize because argocd image updater only supports kustomize and only modifies the kustomize.yaml file with the new image version. 
For the image version alone ArgoCD reffers to the kustomize.yaml file, as it overides the version in the deployment.yaml file.
./k8s/
├── application.yaml
├── configmap.yaml
├── deployment.yaml
├── kustomization.yaml
└── service.yaml

The application file is the ArgoCD Application configs.
Applying the file via kubectl will create an ArgoCD Application in the UI

(((Add git creds for ImageUpdater to commit)))
kubectl -n argocd-image-updater create secret generic git-creds \
  --from-literal=username=someuser \
  --from-literal=password=somepassword

Reffer ├── application.yaml for annotations
